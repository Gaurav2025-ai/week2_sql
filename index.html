üéØ What is a WHERE Clause?

Think of WHERE as a filter that helps you find specific rows in your data.

SELECT columns FROM table WHERE condition;

How it works:

Step What happens Example
1 Look at each row Checking row 1, 2, 3...
2 Test the condition Is price greater than 500?
3 Keep if TRUE ‚úÖ Yes, include this row
4 Skip if FALSE ‚ùå No, skip this row

üîß Comparison Operators

Why learn operators first?
Before we can filter data with WHERE clauses, we need to understand HOW to compare values. Operators are the tools that let us say "find items equal to this" or "greater than that". Think of them as the language we use to describe what we're looking for!

These symbols help you compare values:

=

Equal to

price = 100

Finds exact matches

>

Greater than

price > 500

Values above 500

<

Less than

stock < 10

Values below 10

>=

Greater or equal

rating >= 4.5

4.5 and above

<=

Less or equal

year <= 2000

2000 and below

!=

Not equal

category != 'Phone'

Everything except phones

üìù Understanding Data Types

Important: Different types of data need different formatting in SQL!
Data Type Description How to Write Example
Text/String Words, names, categories Use single quotes ' ' 'Phone', 'Laptop', 'John'
Numbers Prices, quantities, ratings No quotes needed 999, 4.8, 50
Dates Dates and times Use single quotes ' ' '2024-01-15', '2024-12-25'
Boolean True/False values TRUE or FALSE TRUE, FALSE
-- Examples of correct data type usage: WHERE category = 'Phone' -- Text needs quotes WHERE price > 500 -- Numbers don't need quotes WHERE in_stock = TRUE -- Boolean values WHERE launch_date > '2020-01-01' -- Dates need quotes
